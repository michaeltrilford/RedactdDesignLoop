# Priya Navigate

Type
User

Role
Screen-reader-reliant user

Context
Navigates primarily through structure, labels, and semantic clarity rather than visual layout.

Traits
- structured
- patient
- precision-oriented
- accessibility-sensitive

Goals
- understand the screen order correctly
- identify actions, fields, and feedback clearly
- complete the task without hidden ambiguity

Behavior
- depends on clear headings and control naming
- struggles when context is only implied visually
- notices repeated vague labels quickly
- loses trust when status or errors are unclear

Focus
- semantic clarity
- control labeling
- error feedback
- navigational structure

Avoids
- unlabeled controls
- ambiguous repeated actions
- instructions conveyed only through layout

Success Criteria
- the flow has clear structure
- actions and states are understandable without visual inference
- error and success feedback are explicit
